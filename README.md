## Overview
LiftGuessr is an educational library of bodybuilding exercises, with an interactive memorization game and a daily challenge. I have based the concept off of popular webgames such as "Wordle" and "FoodGuessr", using a similar naming style as a nod to the inspirations. This project serves as a showcase of my first foray into JavaScript and into adhering to HTML and CSS design standards and quality. The project also aims to use API services to extend functionality beyond a simple webpage.

## User Experience

#### Target audiences
- **Beginners** - People new to resistance training who want to learn exercise names and what muscles they target.
- **Regular gym-goers** - Those who want to fill gaps in their knowledge or quickly look up an exercise.
- **Trainees and coaches** - Students or instructors building or refreshing their exercise vocabulary.
- **Casual learners** - Anyone who enjoys quick, game-style learning (e.g. Wordle-style daily challenge).

#### User goals
- **Learn** - Quickly find and understand bodybuilding exercises (name, target muscles, etc).
- **Practice** - Test and reinforce knowledge through the memorization game.
- **Compete** - Take on the daily challenge and see how they perform against other players.
- **Use anywhere** - Use the site on phone, tablet, or desktop without a poor experience.

#### User stories
- As a visitor, I want to browse exercises by name or category so that I can learn at my own pace.
- As a visitor, I want to play the memorization game so that I can check what I’ve learned.
- As a visitor, I want a daily challenge so that I have a reason to return and stay engaged.
- As a visitor, I want clear navigation and readable layout so that I can find what I need without confusion.

## Deployment
#### GitHub Pages -
1. Navigate to the GitHub Repo you would like to deploy to Pages
2. Click the **"Settings"** tab (near the top of the page)
3. Click **"Pages"** (in the sidebar menu to the left)
4. Under **"Build and Deployment"** there should be two subheadings - **"Source"** and **"Branch"**.
5. Under **"Source"** select **"Deploy from a branch"** in the dropdown
6. Under **"Branch"** select **"main"** for the branch and **"root"** for the folder
7. Click **"Save"**
